# Seguridad

No publiques vulnerabilidades ni credenciales en una incidencia pública.

Para informar de un problema de seguridad, utiliza la función **Private vulnerability reporting** de GitHub cuando esté habilitada. Si no está disponible, contacta con el mantenedor por un canal privado antes de revelar detalles.

Nunca adjuntes:

- claves privadas;
- JWT completos;
- correos personales;
- copias sin revisar de `.storage`;
- registros que contengan credenciales.
